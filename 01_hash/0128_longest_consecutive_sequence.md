# 128. 最长连续序列 (Longest Consecutive Sequence)

## 题目描述

给定一个未排序的整数数组 `nums`，找出数字连续的最长序列（不要求序列元素在原数组中连续）的长度。

请你设计并实现时间复杂度为 **O(n)** 的算法解决此问题。

**示例：**
```
输入：nums = [100, 4, 200, 1, 3, 2]
输出：4
解释：最长数字连续序列是 [1, 2, 3, 4]。它的长度为 4。
```

## 思路分析

### 为什么不能排序？

排序后很容易找连续序列，但排序是 O(n log n)，题目要求 O(n)。

### 哈希表法 ✅

**核心思想：** 找到每个连续序列的**起点**，然后从起点开始向后数。

**如何判断一个数是连续序列的起点？**
- 如果 `num - 1` 不在数组中，那么 `num` 就是一个连续序列的起点
- 例如：在 `[100, 4, 200, 1, 3, 2]` 中，`1` 是起点（因为 `0` 不在数组中）

**算法步骤：**
1. 将所有数放入一个 HashSet（O(1) 查找）
2. 遍历每个数 `num`：
   - 如果 `num - 1` 在集合中，**跳过**（因为 `num` 不是起点）
   - 如果 `num - 1` 不在集合中，从 `num` 开始，不断检查 `num+1, num+2, ...` 是否在集合中
   - 记录当前连续序列的长度
3. 返回最大长度

**为什么是 O(n)？**
- 虽然有嵌套循环，但每个数最多只被访问两次：一次在外层循环，一次在内层计数
- 只有序列起点才会触发内层循环，且内层循环的总步数不超过 n

## 代码实现

```python
class Solution:
    def longestConsecutive(self, nums: list[int]) -> int:
        # 去重并放入集合，O(1) 查找
        num_set = set(nums)
        max_length = 0
        
        for num in num_set:
            # 只从序列起点开始计数
            if num - 1 not in num_set:
                # num 是一个连续序列的起点
                current = num
                length = 1
                
                # 不断查找下一个连续数
                while current + 1 in num_set:
                    current += 1
                    length += 1
                
                max_length = max(max_length, length)
        
        return max_length
```

## 复杂度分析

- **时间复杂度：** O(n)。虽然看起来有嵌套循环，但内层 while 循环的总执行次数不超过 n
- **空间复杂度：** O(n)，用于存储集合

## 图解示例

```
nums = [100, 4, 200, 1, 3, 2]
num_set = {1, 2, 3, 4, 100, 200}

遍历到 100: 99 不在集合中 → 起点！→ 100, 101? ✗ → 长度 1
遍历到 4:   3 在集合中 → 跳过（4不是起点）
遍历到 200: 199 不在集合中 → 起点！→ 200, 201? ✗ → 长度 1
遍历到 1:   0 不在集合中 → 起点！→ 1→2→3→4→5? ✗ → 长度 4 ✅
遍历到 3:   2 在集合中 → 跳过
遍历到 2:   1 在集合中 → 跳过

结果：4
```

## 关键要点

1. **HashSet 的 O(1) 查找**是实现 O(n) 的关键
2. **只从起点开始计数**的策略，避免了重复计算，保证了整体 O(n)
3. 判断起点的方法非常巧妙：`num - 1` 不在集合中
4. 这道题是"看似 O(n²) 实则 O(n)"的经典案例
