# 1. 两数之和 (Two Sum)

## 题目描述

给定一个整数数组 `nums` 和一个整数目标值 `target`，请你在该数组中找出**和为目标值** `target` 的那**两个**整数，并返回它们的数组下标。

你可以假设每种输入只会对应一个答案，并且你不能使用两次相同的元素。

**示例：**
```
输入：nums = [2,7,11,15], target = 9
输出：[0,1]
解释：因为 nums[0] + nums[1] == 9 ，返回 [0, 1]
```

## 思路分析

### 暴力法（不推荐）
最直观的方法是用两层循环，枚举所有的两数组合，检查它们的和是否等于 target。时间复杂度 O(n²)，太慢。

### 哈希表法（最优解）✅

**核心思想：** 对于每个数 `num`，我们需要找的是 `target - num`。问题变成了：**如何快速判断某个值是否存在于数组中？**

答案就是**哈希表**。哈希表的查找时间复杂度为 O(1)。

**算法步骤：**
1. 创建一个哈希表（字典），存储 `值 -> 下标` 的映射
2. 遍历数组中的每个元素 `num`：
   - 计算 `complement = target - num`
   - 如果 `complement` 在哈希表中，说明找到了答案，返回两个下标
   - 否则，将当前 `num: index` 加入哈希表
3. 注意：我们是**边遍历边存入**，这样可以保证不会使用同一个元素两次

**为什么这样做是正确的？**
- 当我们遍历到 `nums[i]` 时，哈希表中只有 `nums[0]` 到 `nums[i-1]` 的值
- 如果 `target - nums[i]` 在哈希表中找到了，说明之前有一个元素与当前元素的和恰好等于 target
- 这自然保证了两个元素不是同一个

## 代码实现

```python
class Solution:
    def twoSum(self, nums: list[int], target: int) -> list[int]:
        # 哈希表：值 -> 下标
        hash_map = {}
        
        for i, num in enumerate(nums):
            complement = target - num
            
            # 检查 complement 是否已经在哈希表中
            if complement in hash_map:
                return [hash_map[complement], i]
            
            # 将当前元素存入哈希表
            hash_map[num] = i
        
        # 题目保证有解，不会走到这里
        return []
```

## 复杂度分析

- **时间复杂度：** O(n)，只需遍历数组一次，哈希表的查找是 O(1)
- **空间复杂度：** O(n)，最坏情况下需要将所有元素存入哈希表

## 关键要点

1. **哈希表是"空间换时间"的典型应用**：用 O(n) 的空间换来 O(1) 的查找速度
2. **边遍历边存入**是这道题的精髓，避免了使用同一元素两次的问题
3. 这是面试中最常见的入门题，一定要做到 bug-free
